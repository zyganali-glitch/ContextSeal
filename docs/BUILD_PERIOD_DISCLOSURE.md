# Build-period disclosure

ContextSeal was created as a new project during the Build with DataHub: The Agent Hackathon submission period.

## Originality and pre-existing work

The author had prior experience and general product ideas related to agent governance, memory, evidence, and control planes. That experience informed judgment, but no source code, UI, fixture, branded copy, or runtime dependency from a pre-existing personal project was copied into ContextSeal.

## Work created for this repository

- ContextSeal concept, name, and product narrative
- DataHub-native change-certification loop
- Typed request and approval contracts
- Raw MCP evidence normalizer and exact-path collector
- Deterministic impact and risk engines
- Safe dbt migration, test, rollback, and owner-brief generator
- Passport v2 binding and replay/freshness controls
- Operator-protected DataHub read/write/read-back adapter
- Synthetic fixture and native-entity local DataHub seed
- Judge dashboard, tests, CI, Docker path, evidence tooling, skill, and documentation

## Development assistance

AI coding assistants were used for repository analysis, implementation, testing, adversarial review, UI/UX review, and documentation drafting under the author's direction. The author selected the product concept and scope and is responsible for the submitted work and claims.

The official rules permit standard development tools and AI coding assistants. No undisclosed pre-existing project code was incorporated.
