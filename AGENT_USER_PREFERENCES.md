# ContextSeal User Preferences

## Communication and working style

1. Be direct, brutally honest, and evidence-first.
2. Optimize for winning the hackathon, not for writing the safest possible generic roadmap.
3. Prefer concise Turkish-facing summaries, but keep repo governance artifacts readable for English technical review.
4. The user wants maximum leverage upgrades under time and token limits.

## Constraint preferences

1. Assume zero paid new tooling or services unless the user explicitly approves an exception.
2. Free, open-source, or local-first paths are preferred for demo, model, diagram, and video workflows.
3. A living plan with micro-phases is required; vague high-level suggestions are not enough.
